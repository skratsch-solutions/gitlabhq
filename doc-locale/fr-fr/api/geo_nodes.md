Utilisez cette API pour gérer les [nœuds Geo](../administration/geo/_index.md).

Prérequis :

- Vous devez être un administrateur.

## Créer un nœud Geo {#create-a-geo-node}

Crée un nœud Geo.

```plaintext
POST /geo_nodes
```

```shell
curl --request POST \
  --header "PRIVATE-TOKEN: <your_access_token>" \
  --url "https://primary.example.com/api/v4/geo_nodes" \
  -d "name=himynameissomething" \
  -d "url=https://another-node.example.com/"
```

| Attribut                   | Type    | Obligatoire | Description                                                      |
| ----------------------------| ------- | -------- | -----------------------------------------------------------------|
| `primary`                   | boolean | non       | Indique si ce nœud doit être le nœud primaire. La valeur par défaut est false. |
| `enabled`                   | boolean | non       | Indicateur signalant si le nœud Geo est activé. La valeur par défaut est true.    |
| `name`                      | string  | oui      | L'identifiant unique du nœud Geo. Doit correspondre à `geo_node_name` s'il est défini dans `gitlab.rb`, sinon doit correspondre à `external_url` |
| `url`                       | string  | oui      | L'URL visible par l'utilisateur pour le nœud Geo. |
| `internal_url`              | string  | non       | L'URL définie sur le nœud primaire que les nœuds secondaires doivent utiliser pour le contacter. Retourne `url` si non défini. |
| `files_max_capacity`        | entier | non       | Contrôle la simultanéité maximale du remplissage rétroactif LFS/pièces jointes pour ce nœud secondaire. La valeur par défaut est 10. |
| `repos_max_capacity`        | entier | non       | Contrôle la simultanéité maximale du remplissage rétroactif des dépôts pour ce nœud secondaire. La valeur par défaut est 25. |
| `verification_max_capacity` | entier | non       | Contrôle la simultanéité maximale de la vérification des dépôts pour ce nœud. La valeur par défaut est 100. |
| `container_repositories_max_capacity` | entier  | non | Contrôle la simultanéité maximale de la synchronisation des dépôts de conteneurs pour ce nœud. La valeur par défaut est 10. |
| `sync_object_storage`       | boolean | non       | Indicateur signalant si le nœud Geo secondaire doit répliquer les blobs dans Object Storage. La valeur par défaut est false. |
| `selective_sync_type`       | string  | non       | Limiter la synchronisation à des groupes ou des fragments spécifiques uniquement. Valeurs valides : `"namespaces"`, `"shards"` ou `null`. |
| `selective_sync_shards`     | tableau   | non       | Le stockage des dépôts pour les projets synchronisés si `selective_sync_type` == `shards`. |
| `selective_sync_namespace_ids` | tableau | non      | Les ID des groupes à synchroniser, si `selective_sync_type` == `namespaces`. |
| `minimum_reverification_interval` | entier | non | L'intervalle (en jours) pendant lequel la vérification du dépôt est valide. Une fois expiré, il est revérifié. Cela n'a aucun effet lorsqu'il est défini sur un nœud secondaire. |
| `blob_download_timeout`           | entier | non       | Le délai d'expiration (en secondes) pour la réplication des blobs. La valeur par défaut est 28800. Le maximum est 86400. |
