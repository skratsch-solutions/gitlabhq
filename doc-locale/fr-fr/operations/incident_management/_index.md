Pour déterminer si la fonctionnalité répond à vos besoins, consultez les [tickets de bugs ouverts](https://gitlab.com/gitlab-org/gitlab/-/issues/?sort=updated_desc&state=opened&label_name%5B%5D=Category%3AIncident%20Management&label_name%5B%5D=type%3A%3Abug&first_page_size=20).

La gestion des incidents permet aux équipes de développement de trier et de consulter les alertes et les incidents générés par leur application. En faisant remonter les alertes et les incidents là où le code est développé, l'efficacité et la visibilité peuvent être améliorées. Consultez les sections suivantes pour plus d'informations :

- [Intégrez vos outils de surveillance](integrations.md).
- Gérez les [plannings d'astreinte](oncall_schedules.md) et recevez des [notifications](paging.md) pour les alertes déclenchées.
- Triez les [alertes](alerts.md) et les [incidents](incidents.md).
- Informez les parties prenantes avec la [page de statut](status_page.md).
