プランナーエージェントは、GitLabのプロダクト管理および計画ワークフローを支援する特化型AIエージェントです。作業をより効率的に作成、優先順位付け、追跡するのに役立ちます。

The Planner Agentは、作業アイテムの階層（エピック、イシュー、タスク）、マイルストーン、ラベル、ウェイトといったGitLabの計画コンセプトを理解しています。これは、作業アイテムを分析し、優先順位付け戦略を提案し、計画を構成して伝えるのに役立ちます。

トップレベルグループおよびユーザー設定で[Orbit](https://docs.gitlab.com/orbit/)を有効にすると、プランナーエージェントは知識グラフを使用して、関係性、依存関係、およびクロスプロジェクト間の履歴コンテキストに関する質問にも回答できます。

プランナーエージェントは、次のような場合に役立ちます:

- 優先順位付け: RICE、MoSCoW、WSJFなどのフレームワークを適用して、作業アイテムを順位付けする。
- 作業分解: イニシアチブをエピック、機能、ユーザーストーリーに分解する。
- コンテンツの作成: メモ、要件、その他の計画アーティファクトのドラフト作成、または、要求されたときにエピック、イシュー、タスクを直接作成します。
- 依存関係分析: ブロックされている作業を特定し、アイテム間の関係性を把握する。
- コンテンツの編集: 要求されたとき、またはアクションを実行する確認を求めた後に、作業アイテム、ラベル、マイルストーン、その他の属性を更新します。
- To-Do管理: 作業アイテムにTo-Doを追加し、完了としてマークします。
- 計画セッション: スプリント、マイルストーン、四半期計画を整理する。
- ステータスレポート: 進捗状況、リスク、ブロッカーの要約を生成する。
- バックログ管理: 古くなったイシュー、重複しているアイテム、精査が必要なアイテムを特定する。
- 見積もり: 作業アイテムに対する相対的な規模や労力の見積もりを提案する。
- グラフベースのコンテキスト: 類似する過去の作業、クロスプロジェクト間の依存関係、関連する経験を持つコントリビューターを検索します。[Orbit](https://docs.gitlab.com/orbit/)が必要です。

[イシュー583008](https://gitlab.com/gitlab-org/gitlab/-/work_items/583008)でフィードバックを残すことができます。

## Planner Agentを使用する {#use-the-planner-agent}

GitLab UI、VS Code、JetBrains IDEでPlanner Agentを使用できます。

### 最高の結果を得るためのヒント {#tips-for-best-results}

リクエストから最高の結果を得るには:

- URL、フィルター条件、スコープなど、リクエストに関するコンテキストを提供する。
- 関連する場合は、作業アイテムのタイプ（エピック、イシュー、またはタスク）を指定します。
- 使用したい優先順位付けフレームワークがある場合は、それを指定する。
- 概要や更新を求めるときは、対象読者を指定します。たとえば、エンジニアリングチーム、リーダーシップ、またはステークホルダーなどです。
- エージェントにレコメンデーションではなくアクションを実行させたい場合は、「作成」、「更新」、「クローズ」などの明示的な動詞を使用します。
- エージェントの前提がワークフローと一致しない場合は、説明を求めます。

### GitLab UIの場合 {#in-the-gitlab-ui}

前提条件: 

- [基本エージェントをオンにします](_index.md#turn-foundational-agents-on-or-off)。
- オプション。グラフベースのコンテキストを使用するには、トップレベルグループおよびユーザー設定で[Orbitを有効にします](https://docs.gitlab.com/orbit/remote/getting-started/)。

GitLab UIでPlanner Agentを使用するには:

1. 上部のバーで、**検索または移動先**を選択して、プロジェクトまたはグループを見つけます。
1. GitLab Duoサイドバーで、**新しいチャットを追加**（{{< icon name="pencil-square" >}}）を選択します。
1. ドロップダウンリストから**プランナー**を選択します。

   画面右側のGitLab Duoサイドバーに、Chatの会話が表示されます。
1. 計画に関する質問またはリクエストを入力します。

### VS Codeの場合 {#in-vs-code}

前提条件: 

- [基本エージェントをオンにします](_index.md#turn-foundational-agents-on-or-off)。
- [GitLab for VS Code](../../../../editor_extensions/visual_studio_code/setup.md)バージョン6.57.3以降をインストールして設定します。
- [デフォルトのGitLab Duoネームスペース](../../../profile/preferences.md#set-a-default-gitlab-duo-namespace)を設定します。
- オプション。グラフベースのコンテキストを使用するには、トップレベルグループおよびユーザー設定で[Orbitを有効にします](https://docs.gitlab.com/orbit/remote/getting-started/)。

VS CodeでPlanner Agentを使用するには:

1. VS Codeの左サイドバーで、**GitLab Duo Agent Platform**（{{< icon name="duo-agentic-chat" >}}）を選択します。
1. **Chat**タブを選択します。
1. **新しいチャット**（{{< icon name="duo-chat-new" >}}）ドロップダウンリストから、**プランナー**を選択します。
1. 計画に関する質問またはリクエストを入力します。

### JetBrains IDEの場合 {#in-jetbrains-ides}

前提条件: 

- [基本エージェントをオンにします](_index.md#turn-foundational-agents-on-or-off)。
- [JetBrains IDE用のGitLab Duoプラグイン](../../../../editor_extensions/jetbrains_ide/setup.md)バージョン3.11.1以降をインストールして設定します。
- [デフォルトのGitLab Duoネームスペース](../../../profile/preferences.md#set-a-default-gitlab-duo-namespace)を設定します。
- オプション。グラフベースのコンテキストを使用するには、トップレベルグループおよびユーザー設定で[Orbitを有効にします](https://docs.gitlab.com/orbit/remote/getting-started/)。

まず、GitLab Duo Agent Platformを有効にします:

1. JetBrains IDEで、**Settings** > **Tools** > **GitLab Duo**に移動します。
1. **GitLab Duo Agent Platform**で、**Enable GitLab Duo Agent Platform**チェックボックスを選択します。
1. プロンプトが表示されたら、IDEを再起動します。

次に、Planner Agentを使用するには:

1. JetBrains IDEの右側のツールウィンドウバーで、**GitLab Duo Agent Platform**（{{< icon name="duo-agentic-chat" >}}）を選択します。
1. **Chat**タブを選択します。
1. **新しいチャット**（{{< icon name="duo-chat-new" >}}）ドロップダウンリストから、**プランナー**を選択します。
1. 計画に関する質問またはリクエストを入力します。

## プロンプトの例 {#example-prompts}

- 優先順位付け: 
  - 「私のバックログにあるラベル`<label name>`を使用して、RICEフレームワークで作業アイテムの優先順位付けを支援してください。」
  - 「MoSCoWを使用して、顧客への影響に基づいて`<criteria>`の基準を持つこの作業アイテムの機能を分類してください: `<URL>`」
  - 「Q1の戦略的価値によって、これらの作業アイテムをランク付けします: `<URLs>`」
  - 「労力対影響マトリックスを使用して、これらの機能を比較します: `<URLs>`」
  - 「締め切りに間に合わせるために、この作業アイテムの子作業アイテムのうち、どれを現在のスコープから削除すべきですか? `<URL>`」
- 作業分解: 
  - 「このイニシアチブを、提供する必要がある主要な機能に分解します: `<URL>`」
  - 「この作業アイテムを実装するために、どのようなタスクが必要ですか? `<URL>`」
  - 「この機能のMVPバージョンは何ですか? `<URL>`」
  - 「この作業アイテムの機能をどのように順序付けすべきですか? `<URL>`」
  - 「このプロジェクトの段階的アプローチを提案します: `<URL>`」
- ステータスレポート: 
  - 「すべてのサブ作業アイテムを含む、この作業アイテムのステータス更新と進捗レポートを提供します: `<URL>`」
  - 「この作業アイテムの進捗状況のエグゼクティブサマリーを生成します: `<URL>`」
  - 「リーダーシップのために、ブロッカーと軽減計画を要約します: `<URL>`」
  - 「このイニシアチブの健全性に関するステークホルダー向けの更新を作成します: `<URL>`」
- コンテンツの作成:
  - 「目的、成功基準、主要なステークホルダーを含む、このイニシアチブのメモを作成します: `<URL>`」
  - 「APIの要件、データモデル、インテグレーションポイントを含む、この技術要件作業アイテムを作成します: `<URL>`」
  - 「これらの作業アイテム間の関係と順序を説明する依存マップのナラティブを作成します: `<URLs>`」
  - 「潜在的なブロッカーと軽減戦略を特定する、このエピックのリスク評価を生成します: `<URL>`」
  - 「開発時間、テスト、潜在的なブロッカーを含む、この作業アイテムの実装に必要な労力を見積もります: `<URL>`」
- 依存関係分析: 
  - 「スコープを減らすために、この作業アイテムでどの作業を延期すべきですか? `<URL>`」
  - 「新機能と比較して、技術的負債の優先順位付けを手伝ってください。」
- 計画セッション: 
  - 「論理的なリリーステーマに、これらの作業アイテムをグループ化します: `<URLs>`」
  - 「どの作業アイテムが期限を過ぎていますか？」
- バックログ管理: 
  - 「6ヶ月間更新されていない古い作業アイテムを見つけます。」
  - 「このプロジェクト内で重複または類似している作業アイテムを特定してください。」
  - 「私に割り当てられている作業アイテムを表示してください。」
- コンテンツの編集:
  - 「この作業アイテムを完了としてクローズし、何がうまくいったか、何を改善する必要があるかをドキュメント化する新しいレトロスペクティブ作業アイテムを作成します: `<URL>`」
- To-Do管理: 
  - 「作業アイテムにTo-Doを追加: `<URL>`」
  - 「この作業アイテムのすべてのTo-Doを完了としてマーク: `<URL>`」
- グラフベースのコンテキスト（[Orbit](https://docs.gitlab.com/orbit/)が必要）:
  - 「既にリリースされた、これと類似するエピックを検索: `<URL>`」
  - 「この作業アイテムについて、過去に類似するイニシアティブはどのように分解されましたか: `<URL>`」
  - 「このアイデアに関連する作業をすでに誰かが追跡していますか？」
  - 「このエピックに関連する分野で作業したコントリビューターは誰ですか？ `<URL>`」
  - 「この作業アイテムが依存する可能性のある他のクロスプロジェクト間の作業アイテムは何ですか？ `<URL>`」

## 既知の問題 {#known-issues}

- エージェントは作業アイテムを一括で分析できますが、多数のアイテムを含むリクエストの場合、応答品質が低下する可能性があります。
- エージェントは、ディスカッション履歴が長い作業アイテムのコメントに確実にアクセスできません。約100エントリを超えるコメント履歴は、不完全である可能性があります。
- エージェントに、存在しない値に作業アイテムのステータスを更新するように依頼した場合、エージェントは、変更を適用せずに更新が成功したと誤って報告する可能性があります。
- エージェントは作業アイテム間のリンクされたアイテムの関係を作成できません。ただし、エージェントに、`/relate`、`/blocks`、または`/blocked_by`クイックアクションを使用してコメントを追加するように依頼できます。回避策として。
