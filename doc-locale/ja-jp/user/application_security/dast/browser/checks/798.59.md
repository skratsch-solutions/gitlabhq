## 説明 {#description}

レスポンスボディに、Heroku APIキーまたはアプリケーション認可トークンのパターンに一致するコンテンツが特定されました。APIキーと認可トークンは、ユーザーまたはアカウントに代わってAPIコールを実行するために使用できます。これらのトークンにアクセスできる悪意のあるアクターは、Heroku APIプラットフォームと、すべてのデプロイ済みアプリケーションにアクセスできます。この値を露出すると、攻撃者がこのトークンによって付与されたすべてのリソースへのアクセスを取得する可能性があります。

## 修復 {#remediation}

キー漏洩に関するセキュリティインシデントの対処に関する一般的なガイダンスについては、GitLabドキュメントの[認証情報のインターネットへの露出](../../../../../security/responding_to_security_incidents.md#credential-exposure-to-public-internet)を参照してください。

特定されたユーザーのAPIキーを再生成するには:

- アカウントにサインインし、<https://dashboard.heroku.com/account>にアクセスしてください。
- 「APIキー」セクションで、「Regenerate API Key」を選択します。
- プロンプトが表示されたら、「Regenerate API Key」ダイアログで「Regenerate API Key」を選択します。

アプリケーション認可トークンを再生成するには:

- アカウントにサインインし、<https://dashboard.heroku.com/account/applications>にアクセスしてください。
- 「Authorizations」セクションで、特定されたトークンを含む登録済みの認可を見つけます。
- 鉛筆アイコンを選択します。
- 「Regenerate token」を選択します。

APIキーの詳細については、[APIキーの生成に関するFAQ](https://help.heroku.com/PBGP6IDE/how-should-i-generate-an-api-key-that-allows-me-to-use-the-heroku-platform-api)を参照してください。Herokuには、アプリケーション認可トークンに関するドキュメントがありません。
