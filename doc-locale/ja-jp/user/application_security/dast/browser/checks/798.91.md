## 説明 {#description}

レスポンスボディに、PlanetScale APIサービストークンのパターンに一致するコンテンツが識別されました。サービストークンは、許可されたスコープに応じて作成され、権限が割り当てられます。サービストークンへのアクセス権を持つ悪意のある攻撃者には、このサービストークンに割り当てられたのと同じ権限が付与されます。この値を露出すると、攻撃者がこのトークンによって付与されたすべてのリソースへのアクセスを取得する可能性があります。

## 修復 {#remediation}

キー漏洩に関するセキュリティインシデントの対処に関する一般的なガイダンスについては、GitLabドキュメントの[認証情報のインターネットへの露出](../../../../../security/responding_to_security_incidents.md#credential-exposure-to-public-internet)を参照してください。

サービストークンを失効するには:

- PlanetScaleアカウントにサインインして、<https://app.planetscale.com/>にアクセスしてください。
- 左側のメニューから「Settings」を選択します。
- 「設定」の下にある「Service tokens」を選択します。
- 特定されたセキュリティトークンを見つけて、その名前を選択します。
- 組織のアクセス権、パーミッション、およびスコープを記録します。
- 右上隅にある「Delete service token」を選択します。
- プロンプトが表示されたら、「Delete service token」ダイアログで「Delete」を選択します。

詳細については、[PlanetScaleのサービストークンに関するドキュメント](https://planetscale.com/docs/concepts/service-tokens)を参照してください。
