ここに示されている認定と指定は、[GitLabのAWSパートナーページ](https://partners.amazonaws.com/partners/001E0000018YWFfIAO/GitLab,%20Inc.)で検証できます。

すべてのAWSパートナー資格は、広範なチェックリストの提出と検証、およびAWSが資格を付与するかどうかを決定するために利用する裏付けとなる証拠の提出を必要とします。

## DevOpsソフトウェア/ISVコンピテンシー {#devops-software--isv-competency}

このコンピテンシーは、GitLabがAWSと連携してAWS上で動作するDevOpsソリューションを提供することを検証します。[AWSプログラム情報](https://aws.amazon.com/devops/partner-solutions/)

## DevSecOpsスペシャリティカテゴリ {#devsecops-specialty-category}

DevSecOps資格は、DevOpsソフトウェアコンピテンシーのカテゴリであり、GitLabが組織のDevSecOps成熟度目標達成を支援する上で重要なソリューションであることを示しています。GitLabは、この指定が与えられる前に、これらの追加資格を満たしているか審査されました。[AWSプログラム情報](https://aws.amazon.com/blogs/apn/aws-devops-competency-expands-to-include-devsecops-category/) [GitLab発表](https://about.gitlab.com/blog/aws-devsecops-competency-partner/)

## 公共機関パートナー {#public-sector-partner}

この指定は、GitLabがAWS公共機関顧客と連携する資格があると認められたことを示しています。GitLabには、公共機関特有のニーズに対応する専門の組織があります。[AWSプログラム情報](https://aws.amazon.com/partners/programs/public-sector/)

## AWS Graviton {#aws-graviton}

GitLabインスタンスとRunnerは、AWS Gravitonでテストされ、動作することが確認されています。Amazon Linux向けに、ARMアーキテクチャ用のYUMパッケージをメンテナンスしています。[AWSプログラム情報](https://aws.amazon.com/ec2/graviton/partners/)

## Amazon Linux対応 {#amazon-linux-ready}

GitLabインスタンスとRunnerは、Amazon Linux 2および2023で検証済みです。これには、両方に対応するYUMパッケージとパッケージリポジトリ、およびパッケージ化前の両方に対応する2300を超えるCIテストが含まれます。[AWSプログラム情報](https://aws.amazon.com/amazon-linux/partners/)

## AWS Marketplace販売者 {#aws-marketplace-seller}

GitLabはMarketplace販売者であり、AWS Marketplaceを通じて購入およびデプロイできます。[AWSプログラム情報](https://aws.amazon.com/marketplace/partners/management-tour)
