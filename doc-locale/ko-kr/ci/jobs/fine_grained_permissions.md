세밀한 권한을 사용하여 제한된 REST API 엔드포인트 집합에 대한 액세스를 명시적으로 허용할 수 있습니다. 이러한 권한은 지정된 프로젝트의 CI/CD 작업 토큰에 적용됩니다.

[이슈 519575](https://gitlab.com/gitlab-org/gitlab/-/issues/519575)에서 피드백을 공유하세요.

## 작업 토큰 허용 목록에 세분화된 권한 추가 {#add-fine-grained-permissions-to-the-job-token-allowlist}

전제 조건:

- 프로젝트에 대해 Maintainer 또는 Owner 역할이 필요합니다.
- 프로젝트에서 세분화된 권한 사용을 활성화해야 합니다.

작업 토큰 허용 목록의 그룹 및 프로젝트에 세분화된 권한을 추가할 수 있습니다. 이를 통해 작업 토큰을 사용하여 특정 프로젝트 리소스에 액세스하고 이러한 그룹 및 프로젝트에서 사용할 수 있는 리소스를 보다 정확하게 제어할 수 있습니다.

작업 토큰 허용 목록의 그룹 또는 프로젝트에 세분화된 권한을 추가하려면:

1. 상단 표시줄에서 **검색 또는 이동**을 선택하고 프로젝트를 찾습니다.
1. **설정** > **CI/CD**를 선택합니다.
1. **작업 토큰 권한**을 확장합니다.
1. **CI/CD 작업 토큰 허용 목록** 섹션에서 **추가**를 선택하세요.
1. 드롭다운 목록에서 **그룹 또는 프로젝트**를 선택하세요.
1. 기존 그룹 또는 프로젝트의 경로를 입력하세요.
1. **세분화된 권한**을 선택하세요.
1. 사용 가능한 [API 엔드포인트](#available-api-endpoints)에 권한을 부여하세요.
1. **추가**를 선택하세요.

GitLab이 지정된 권한과 함께 그룹 또는 프로젝트를 작업 토큰 허용 목록에 추가합니다. 그룹 또는 프로젝트는 이제 현재 프로젝트의 허용된 리소스에 액세스할 수 있습니다.

## 사용 가능한 API 엔드포인트 {#available-api-endpoints}

CI/CD 작업 토큰은 다음 REST API 엔드포인트에 액세스할 수 있습니다:

### 배포 엔드포인트 {#deployments-endpoints}

| 권한 | API 엔드포인트 | 권한 이름 | 범위 |
| ---------- | ------------ | --------------- | ----- |
| 배포와 관련된 모든 머지 리퀘스트 나열 | `GET /projects/:id/deployments/:deployment_id/merge_requests` | `READ_DEPLOYMENTS` | 읽기 |
| 모든 프로젝트 배포 나열 | `GET /projects/:id/deployments` | `READ_DEPLOYMENTS` | 읽기 |
| 배포 검색 | `GET /projects/:id/deployments/:deployment_id` | `READ_DEPLOYMENTS` | 읽기 |
| 차단된 배포 승인 또는 거부 | `POST /projects/:id/deployments/:deployment_id/approval` | `ADMIN_DEPLOYMENTS` | 읽기 및 쓰기 |
| 배포 생성 | `POST /projects/:id/deployments` | `ADMIN_DEPLOYMENTS`, `ADMIN_ENVIRONMENTS` | 읽기 및 쓰기 |
| 배포 삭제 | `DELETE /projects/:id/deployments/:deployment_id` | `ADMIN_DEPLOYMENTS` | 읽기 및 쓰기 |
| 배포 업데이트 | `PUT /projects/:id/deployments/:deployment_id` | `ADMIN_DEPLOYMENTS` | 읽기 및 쓰기 |
