## Geo 사이트 생성 {#create-a-geo-site}

{{< history >}}

- `blob_download_timeout`은 GitLab 18.10에서 [도입됨](https://gitlab.com/gitlab-org/gitlab/-/work_items/569919)되었습니다.

{{< /history >}}

Geo 사이트를 생성합니다.

```plaintext
POST /geo_sites
```

```shell
curl --header "PRIVATE-TOKEN: <your_access_token>" \
     --url "https://primary.example.com/api/v4/geo_sites" \
     --request POST \
     -d "name=himynameissomething" \
     -d "url=https://another-node.example.com/"
```

| 특성                             | 유형    | 필수 | 설명                                                                                                                                            |
|---------------------------------------|---------|----------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| `primary`                             | 부울 | 아니오       | 이 사이트가 기본 사이트인지 여부를 지정합니다. 기본값은 false입니다.                                                                                     |
| `enabled`                             | 부울 | 아니오       | Geo 사이트가 활성화되었는지 나타내는 플래그입니다. 기본값은 true입니다.                                                                                          |
| `name`                                | 문자열  | 예      | Geo 사이트의 고유 식별자입니다. `geo_node_name`이 `gitlab.rb`에 설정된 경우 일치해야 하며, 그렇지 않으면 `external_url`과(와) 일치해야 합니다.                 |
| `url`                                 | 문자열  | 예      | Geo 사이트의 사용자 대면 URL입니다.                                                                                                                  |
| `internal_url`                        | 문자열  | 아니오       | 기본 사이트에 정의된 URL로, 보조 사이트가 이를 사용하여 연결합니다. 설정되지 않은 경우 `url`을(를) 반환합니다.                                           |
| `files_max_capacity`                  | 정수 | 아니오       | 이 보조 사이트의 LFS/첨부 파일 백필의 최대 동시성을 제어합니다. 기본값은 10입니다.                                                    |
| `repos_max_capacity`                  | 정수 | 아니오       | 이 보조 사이트의 리포지토리 백필의 최대 동시성을 제어합니다. 기본값은 25입니다.                                                        |
| `verification_max_capacity`           | 정수 | 아니오       | 이 사이트의 리포지토리 검증의 최대 동시성을 제어합니다. 기본값은 100입니다.                                                             |
| `container_repositories_max_capacity` | 정수 | 아니오       | 이 사이트의 컨테이너 리포지토리 동기화의 최대 동시성을 제어합니다. 기본값은 10입니다.                                                            |
| `sync_object_storage`                 | 부울 | 아니오       | 보조 Geo 사이트가 객체 저장소의 BLOB을 복제해야 하는지를 나타내는 플래그입니다. 기본값은 false입니다.                                                 |
| `selective_sync_type`                 | 문자열  | 아니오       | 동기화를 특정 그룹 또는 샤드로만 제한합니다. 유효한 값: `"namespaces"`, `"shards"` 또는 `null`입니다.                                                  |
| `selective_sync_shards`               | 배열   | 아니오       | `selective_sync_type` == `shards`인 경우 동기화된 프로젝트의 리포지토리 저장소입니다.                                                                   |
| `selective_sync_namespace_ids`        | 배열   | 아니오       | `selective_sync_type` == `namespaces`인 경우 동기화해야 할 그룹의 ID입니다.                                                                     |
| `minimum_reverification_interval`     | 정수 | 아니오       | 리포지토리 검증이 유효한 간격(일)입니다. 만료되면 다시 검증됩니다. 보조 사이트에 설정했을 때는 영향을 주지 않습니다. |
| `blob_download_timeout`               | 정수 | 아니오       | BLOB 복제의 시간 초과(초)입니다. 기본값은 28800입니다. 최대값은 86400입니다.                                                                    |

예제 응답:

```json
{
  "id": 3,
  "name": "Test Site 1",
```

## Geo 사이트 복구 {#repair-a-geo-site}

기본 또는 보조 Geo 사이트 간의 OAuth 동기화 문제가 발생한 경우 Geo 사이트의 OAuth 인증을 복구합니다. 이 경우 다음 메시지가 표시될 수 있습니다:

```plaintext
There are no OAuth application defined for this Geo node.
```

```plaintext
POST /geo_sites/:id/repair
```
