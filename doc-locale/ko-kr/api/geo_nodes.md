> [!warning]
> Geo 노드 API는 GitLab 16.0에서 [지원 중단되었으며](https://gitlab.com/gitlab-org/gitlab/-/issues/369140) API의 v5에서 제거될 예정입니다. 대신 [Geo Sites API](geo_sites.md)를 사용합니다. 이 변경 사항은 주요 변경 사항입니다.

이 API를 사용하여 [리포지토리](../administration/geo/_index.md)를 관리합니다.

전제 조건:

- 관리자(administrator) 권한이 있어야 합니다.

## Geo 노드 생성 {#create-a-geo-node}

Geo 노드를 생성합니다.

```plaintext
POST /geo_nodes
```

```shell
curl --request POST \
  --header "PRIVATE-TOKEN: <your_access_token>" \
  --url "https://primary.example.com/api/v4/geo_nodes" \
  -d "name=himynameissomething" \
  -d "url=https://another-node.example.com/"
```

| 속성                   | 유형    | 필수 | 설명                                                      |
| ----------------------------| ------- | -------- | -----------------------------------------------------------------|
| `primary`                   | 부울 | 아니요       | 이 노드가 기본 노드인지 여부를 지정합니다. 기본값은 거짓입니다. |
| `enabled`                   | 부울 | 아니요       | Geo 노드가 활성화되었는지 여부를 나타내는 플래그입니다. 기본값은 참입니다.    |
| `name`                      | 문자열  | 예      | Geo 노드의 고유 식별자입니다. `geo_node_name`이 `gitlab.rb`에 설정된 경우 일치해야 하고, 그렇지 않으면 `external_url`과 일치해야 합니다. |
| `url`                       | 문자열  | 예      | Geo 노드에 대한 사용자 대면 URL입니다. |
| `internal_url`              | 문자열  | 아니요       | 기본 노드에서 정의한 URL로, 보조 노드가 연락하는 데 사용해야 합니다. 설정하지 않으면 `url`을 반환합니다. |
| `files_max_capacity`        | 정수 | 아니요       | 이 보조 노드에 대한 LFS/첨부 파일 백필의 최대 동시성을 제어합니다. 기본값은 10입니다. |
| `repos_max_capacity`        | 정수 | 아니요       | 이 보조 노드에 대한 리포지토리 백필의 최대 동시성을 제어합니다. 기본값은 25입니다. |
| `verification_max_capacity` | 정수 | 아니요       | 이 노드에 대한 리포지토리 검증의 최대 동시성을 제어합니다. 기본값은 100입니다. |
| `container_repositories_max_capacity` | 정수  | 아니요 | 이 노드에 대한 컨테이너 리포지토리 동기화의 최대 동시성을 제어합니다. 기본값은 10입니다. |
| `sync_object_storage`       | 부울 | 아니요       | 보조 Geo 노드가 Object Storage의 blob을 복제해야 하는지 여부를 나타내는 플래그입니다. 기본값은 거짓입니다. |
| `selective_sync_type`       | 문자열  | 아니요       | 동기화를 특정 그룹 또는 분할만으로 제한합니다. 유효한 값: `"namespaces"`, `"shards"`, 또는 `null`입니다. |
| `selective_sync_shards`     | 배열   | 아니요       | `selective_sync_type` == `shards`인 경우 동기화된 프로젝트의 리포지토리 스토리지입니다. |
| `selective_sync_namespace_ids` | 배열 | 아니요      | `selective_sync_type` == `namespaces`인 경우 동기화해야 할 그룹의 ID입니다. |
| `minimum_reverification_interval` | 정수 | 아니요 | 리포지토리 검증이 유효한 간격(일)입니다. 만료되면 다시 검증됩니다. 보조 노드에서 설정할 때는 효과가 없습니다. |
| `blob_download_timeout`           | 정수 | 아니요       | blob 복제의 타임아웃(초)입니다. 기본값은 28800입니다. 최댓값은 86400입니다. |

응답 예시:

```json
{
  "id": 3,
  "name": "Test Node 1",
```
