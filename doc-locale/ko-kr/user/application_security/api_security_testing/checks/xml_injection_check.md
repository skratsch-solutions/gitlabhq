## 설명 {#description}

XML 직렬화/인젝션 취약점을 확인합니다.

## 수정 {#remediation}

XML 인젝션은 XML 애플리케이션 또는 서비스의 논리를 조작하거나 손상시키는 데 사용되는 공격 기법입니다. 의도하지 않은 XML 콘텐츠 및/또는 구조를 XML 메시지에 주입하면 애플리케이션의 의도된 논리를 변경할 수 있습니다. 또한 XML 인젝션은 악의적 콘텐츠를 결과 메시지/문서에 삽입할 수 있습니다.
