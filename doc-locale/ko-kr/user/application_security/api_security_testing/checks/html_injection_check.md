## 설명 {#description}

문자열을 지원하는 모든 필드로의 HTML 주입을 통해 XSS를 확인합니다. 여기에는 경로, 쿼리, 헤더와 같은 HTTP 요청의 일부와 XML 필드, JSON 필드 등과 같은 본문 매개 변수도 포함됩니다. 감지는 알려진 HTML 지원 필드로 주입된 값에 대한 응답을 모니터링하여 수행됩니다.

## 수정 {#remediation}

크로스사이트 스크립팅(XSS)은 공격자가 제공한 코드를 사용자의 브라우저 인스턴스에 입력하는 공격 기법입니다. 브라우저 인스턴스는 표준 웹 브라우저 클라이언트이거나, WinAmp 내의 브라우저, RSS 리더, 이메일 클라이언트 등의 소프트웨어 제품에 포함된 브라우저 객체일 수 있습니다. 코드는 일반적으로 HTML/JavaScript로 작성되지만, VBScript, ActiveX, Java, Flash 또는 기타 브라우저 지원 기술로 확장될 수도 있습니다.

공격자가 사용자의 브라우저를 자신의 코드를 실행하도록 하면, 코드는 호스팅 웹사이트의 보안 컨텍스트(또는 영역) 내에서 실행됩니다. 이러한 수준의 권한으로 코드는 브라우저에서 액세스할 수 있는 모든 민감한 데이터를 읽고, 수정하고, 전송할 수 있습니다. 크로스사이트 스크립팅된 사용자는 계정이 탈취되거나(쿠키 도용), 브라우저가 다른 위치로 리디렉션되거나, 방문 중인 웹사이트에서 제공하는 거짓 콘텐츠를 보게 될 수 있습니다. 크로스사이트 스크립팅 공격은 본질적으로 사용자와 웹사이트 간의 신뢰 관계를 훼손합니다. 파일 시스템에서 콘텐츠를 로드하는 브라우저 객체 인스턴스를 사용하는 애플리케이션은 로컬 머신 영역에서 코드를 실행하여 시스템 타협을 허용할 수 있습니다.

크로스사이트 스크립팅 공격에는 비지속적, 지속적, DOM 기반의 세 가지 유형이 있습니다.

비지속적 공격 및 DOM 기반 공격은 사용자가 악의적인 코드가 포함된 특별히 제작된 링크를 방문하거나, 악의적인 웹 페이지를 방문하여 웹 양식을 포함하고 있으며, 취약한 사이트에 게시하면 공격을 시작하도록 요구합니다. 악의적인 양식을 사용하는 것은 종종 취약한 리소스가 HTTP POST 요청만 허용할 때 발생합니다. 이러한 경우 양식을 자동으로 제출할 수 있으며, 피해자의 지식 없이(예: JavaScript 사용) 제출됩니다. 악의적인 링크를 클릭하거나 악의적인 양식을 제출하면, XSS 페이로드가 다시 입력되고 사용자의 브라우저에서 해석되어 실행됩니다. 거의 임의의 요청(GET 및 POST)을 보내는 또 다른 기법은 Adobe Flash와 같은 임베디드 클라이언트를 사용하는 것입니다.

지속적 공격은 악의적인 코드가 일정 기간 동안 저장되는 웹사이트에 제출될 때 발생합니다. 공격자의 선호하는 대상의 예는 메시지 게시판 게시물, 웹메일 메시지, 웹 채팅 소프트웨어를 자주 포함합니다. 의심하지 않는 사용자는 추가 사이트/링크(예: 공격자 사이트 또는 이메일을 통해 전송된 악의적인 링크)와 상호 작용할 필요가 없으며, 코드를 포함하는 웹페이지를 보기만 하면 됩니다.
