계획 워크플로우에 맞게 작업 항목 유형을 커스터마이즈합니다. 새 유형을 이름과 아이콘으로 만들고, 프로젝트에서 사용 가능한 유형을 제어합니다. 작업 항목 유형 구성이 모든 프로젝트에 적용됩니다.

새 유형은 프로젝트 수준에서만 사용 가능합니다. 이들의 위젯과 계층 제한 사항은 이슈와 일치합니다. 새 유형을 [사용자 지정 필드](custom_fields.md) 및 [상태 수명 주기](status.md)와 연결할 수 있습니다. 작업 항목 유형으로 필터링하고 [저장된 보기](saved_views.md) 및 이슈 보드를 사용할 수 있습니다.

## 제한 및 명명 규칙 {#limits-and-naming-rules}

GitLab에서 제공하는 유형을 포함하여 최상위 그룹 또는 조직에 최대 40개의 작업 항목 유형을 보유할 수 있습니다. 유형 이름은 네임스페이스 또는 조직 내에서 고유해야 하며 최대 48자입니다. 유형은 아카이브된 유형 및 비활성화된 유형을 포함하여 다른 유형과 같은 이름을 공유할 수 없습니다. 취약성, 머지 리퀘스트, 커밋, 파이프라인, alert, review, diff, report 및 sha는 사용할 수 없는 예약된 이름입니다.

유형의 이름을 바꾸면(예: `Feature`에서 `Enhancement`로), 원래 이름을 새 유형에 사용할 수 있습니다. 해당 이름이 사용 중이 아니면 유형의 이름을 원래 이름으로 바꿀 수 있습니다.

## 작업 항목 유형 상태 {#work-item-type-states}

각 작업 항목 유형은 다음 상태 중 하나를 갖습니다:

| 상태 | 설명 |
|-------|-------------|
| 활성화됨 | 기본 상태입니다. 유형을 사용하여 작업 항목을 만들 수 있으며 필터에 표시됩니다. |
| 비활성화됨 | 유형을 사용하여 새 작업 항목을 만들 수 없으며 필터에 표시되지 않습니다. 비활성화된 유형의 이름을 바꾸고 아이콘을 변경할 수 있습니다. 해당 유형의 기존 작업 항목은 영향을 받지 않습니다. |
| 아카이브됨 | 유형이 모든 목록, 필터 및 생성 플로우에서 제거됩니다. 최상위 그룹 또는 조직 수준에서만 아카이브된 유형을 보고 아카이빙을 해제할 수 있습니다. 아카이브된 유형의 이름을 바꾸거나 아이콘을 변경할 수 없습니다. |
| 잠김 | 유형은 특정 GitLab 기능과 연결되어 있으며 이름을 바꾸거나, 비활성화하거나, 아카이브할 수 없습니다. 예를 들어, Ticket, Incident, 에픽 및 Task 유형은 잠겨 있습니다. |

> [!note]
> 에픽은 프로젝트 수준에서 작업 항목 유형으로 표시되지만 에픽이 그룹 수준에서만 사용 가능하므로 프로젝트의 경우 비활성화됩니다.

## 작업 항목 유형 만들기 {#create-a-work-item-type}

특정 카테고리의 작업을 나타내기 위해 이름과 아이콘을 사용하여 작업 항목 유형을 만듭니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.

작업 항목 유형을 만들려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업 항목 유형** 섹션에서 **새 유형**을 선택합니다.
1. 유형의 이름을 입력합니다.
1. 아이콘을 선택합니다.
1. **저장**을 선택합니다.

## 작업 항목 유형 편집 {#edit-a-work-item-type}

기존 작업 항목 유형의 이름 또는 아이콘을 업데이트합니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.
- 유형은 잠기거나 아카이브되지 않아야 합니다.

작업 항목 유형을 편집하려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업 항목 유형** 섹션에서 편집할 유형을 찾습니다.
1. **이름과 아이콘 수정**을 선택합니다.
1. 이름, 아이콘 또는 둘 다를 업데이트합니다.
1. **저장**을 선택합니다.

> [!note]
> 특정 GitLab 기능과 연결된 유형은 편집할 수 없습니다. 예를 들어, Ticket, Incident, 에픽 및 Task 유형은 잠겨 있으며 이름을 바꾸거나 아이콘을 변경할 수 없습니다.

## 작업 항목 유형 아카이브 {#archive-a-work-item-type}

작업 항목 유형을 아카이브하여 모든 목록, 필터 및 생성 플로우에서 제거합니다. 아카이브된 유형은 시스템에 남아 있으며, 해당 유형의 기존 작업 항목은 영향을 받지 않습니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.

작업 항목 유형을 아카이브하려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업 항목 유형** 섹션에서 아카이브할 유형을 찾습니다.
1. **아카이브**를 선택합니다.
1. 확인 대화 상자에서 **아카이브**를 선택합니다.

아카이브된 유형은 **아카이빙됨** 탭의 최상위 그룹 또는 조직 수준에서만 표시됩니다. 하위 그룹 또는 프로젝트 수준에서는 표시되지 않습니다.

## 작업 항목 유형 아카이빙 해제 {#unarchive-a-work-item-type}

아카이브된 작업 항목 유형을 복원하여 다시 사용 가능하게 합니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.
- 활성 유형의 총 개수는 최대 40개 이하여야 합니다.

작업 항목 유형의 아카이빙을 해제하려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업 항목 유형** 섹션에서 **아카이빙됨** 탭을 선택합니다.
1. 복원할 유형을 찾습니다.
1. **아카이빙 해제**를 선택합니다.
1. 확인 대화 상자에서 **아카이빙 해제**를 선택합니다.

## 유형 가용성 제어 {#type-availability-controls}

프로젝트에서 사용 가능한 작업 항목 유형을 제어합니다. 유형 가용성은 세 가지 계층을 가집니다:

- **프로젝트내의 유형 커스텀화**:  프로젝트가 유형 가용성을 커스터마이즈하도록 허용하거나 방지하는 최상위 그룹 또는 조직의 스위치입니다. 기본적으로 비활성화됨.
- **Per-type availability for all projects**:  모든 하위 프로젝트에서 특정 유형을 활성화하거나 비활성화합니다.
- **Per-project availability**:  개별 프로젝트는 자체 범위에 대해 유형을 활성화하거나 비활성화할 수 있습니다.

### 프로젝트에서 유형 커스터마이제이션 허용 {#allow-type-customization-in-projects}

프로젝트에서 사용 가능한 유형을 커스터마이즈하는 기능을 제어합니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.

유형 커스터마이제이션을 전환하려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **프로젝트내의 유형 커스텀화** 섹션에서 **사용** 또는 **사용 안 함**을 선택합니다.

유형 커스터마이제이션을 비활성화하면 이전 가시성 설정에 관계없이 모든 프로젝트에서 모든 유형이 활성화된 것으로 처리됩니다. 프로젝트별 재정의는 보존되지만 무시됩니다. 유형 커스터마이제이션을 다시 활성화하면 이전 구성이 복원됩니다.

### 모든 프로젝트에 대해 유형 활성화 또는 비활성화 {#enable-or-disable-a-type-for-all-projects}

모든 하위 프로젝트에서 특정 유형의 가용성을 한 번에 제어합니다. 이렇게 하면 해당 유형에 대한 모든 프로젝트별 재정의가 제거됩니다.

전제 조건:

- GitLab.com에서:  최상위 그룹에 대해 최소한 관리자 역할이 있어야 합니다.
- GitLab Self-Managed에서:  인스턴스 관리자이거나 조직 소유자여야 합니다.

모든 프로젝트에 대해 유형을 활성화하거나 비활성화하려면:

1. 작업 항목 설정으로 이동합니다:
   - GitLab.com에서:  상단 표시줄에서 **검색 또는 이동**을 선택하고 최상위 그룹을 찾습니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
   - GitLab Self-Managed에서:  오른쪽 위 모서리에서 **운영자**를 선택합니다. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업 항목 유형** 섹션에서 유형을 찾습니다.
1. 유형 작업 메뉴에서 **모든 프로젝트 활성화** 또는 **모든 프로젝트 비활성화**를 선택합니다.

### 프로젝트에서 유형 활성화 또는 비활성화 {#enable-or-disable-a-type-in-a-project}

특정 프로젝트에서 사용 가능한 유형을 제어합니다.

전제 조건:

- 프로젝트에 대해 최소한 관리자 역할이 있어야 합니다.

프로젝트에서 유형을 활성화하거나 비활성화하려면:

1. 상단 표시줄에서 **검색 또는 이동**을 선택하고 프로젝트를 찾으세요.
1. 왼쪽 사이드바에서 **설정** > **작업 항목**을 선택합니다.
1. **작업항목 아이템 유형 활성화** 섹션에서 유형을 찾습니다.
1. 유형을 활성화하거나 비활성화합니다.

프로젝트에서 유형을 비활성화하면 해당 프로젝트에서 해당 유형을 사용하여 새 작업 항목을 만들 수 없습니다. 해당 유형의 기존 작업 항목은 영향을 받지 않습니다.
