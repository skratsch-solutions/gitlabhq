> [!note]
> 저희는 Geo 문서, 사용자 인터페이스 및 명령을 업데이트하고 있습니다. 모든 페이지가 아직 이러한 정의를 따르지는 않습니다.

이는 Geo의 모든 측면을 설명하기 위해 정의된 용어입니다. 명확하게 정의된 용어 집합을 사용하면 효율적으로 소통할 수 있으며 혼동을 피할 수 있습니다. 이 페이지의 언어는 보편적이고 최대한 간단하기를 목표로 합니다.

## 복제기 용어 {#replicator-terms}

Geo는 복제기를 사용하여 주 사이트와 보조 사이트 간의 개별 GitLab 구성 요소의 데이터를 복제합니다. 이들은 이러한 구성 요소의 개별 [데이터 타입](replication/datatypes.md#data-types)을 처리하고 확인하는 방법을 정의합니다. 예를 들어 GitLab 컨테이너 레지스트리의 데이터는 CI 작업 아티팩트와 다르게 처리되어야 합니다. 일부 구성 요소는 잠재적으로 다르게 이름이 지정된 하나 이상의 복제기를 가질 수 있습니다. 따라서 다음 표는 복제기 이름과 해당 복제기가 속한 GitLab 구성 요소를 설명합니다.

동일한 복제기 이름은 관리자 영역의 Geo 섹션이나 Geo 관련 콘솔 명령을 사용할 때도 표시됩니다.

| Geo 복제기 이름            | GitLab 구성 요소 이름                  |
|--------------------------------|----------------------------------------|
| CI 보안 파일                | CI 보안 파일                        |
| 컨테이너 리포지토리         | 컨테이너 레지스트리                     |
| 종속성 프록시 Blob         | 종속성 프록시 이미지                |
| 종속성 프록시 매니페스트     | 종속성 프록시 이미지                |
| 설계 관리 리포지토리 | 프로젝트 설계 리포지토리             |
| 그룹 위키 리포지토리        | 그룹 위키 리포지토리                  |
| CI 작업 아티팩트               | CI 작업 아티팩트                       |
| LFS 객체                    | LFS 객체                            |
| 머지 리퀘스트 Diffs            | 외부 머지 리퀘스트 diffs           |
| 패키지 파일                  | 패키지 레지스트리                       |
| 페이지 배포              | 페이지                                  |
| 파이프라인 아티팩트             | 파이프라인 아티팩트                     |
| 프로젝트 리포지토리           | 프로젝트 리포지토리                     |
| 프로젝트 위키 리포지토리      | 프로젝트 위키 리포지토리                |
| 스니펫 리포지토리           | 개인 스니펫 및 프로젝트 스니펫 |
| Terraform 상태 버전       | 버전이 지정된 Terraform 상태              |
| 업로드                        | 사용자 업로드                           |
