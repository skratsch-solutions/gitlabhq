GitLab은 GitLab 인스턴스의 사용자에게 두 가지 유형의 전체 알림 메시지를 표시할 수 있습니다:

- 배너
- 알림

전체 알림 메시지는 [전체 알림 메시지 API](../api/broadcast_messages.md)를 사용하여 관리할 수 있습니다.

> [!warning]
> 전체 알림 메시지는 대상 지정 설정에 관계없이 API를 통해 공개적으로 액세스할 수 있습니다. 민감하거나 기밀 정보를 포함하지 마세요. 또한 전체 알림 메시지를 사용하여 특정 그룹이나 프로젝트에 비공개 정보를 전달하지 마세요.

## 배너 {#banners}

배너는 페이지의 상단에 표시되며, 선택적으로 Git 리모트 응답으로 명령줄에 표시됩니다.

![환영 메시지를 표시하는 전체 알림 메시지 배너입니다.](img/broadcast_messages_banner_v17_7.png)

```shell
$ git push
...
remote:
remote: **Welcome to GitLab** :wave:
remote:
...
```

한 번에 두 개 이상의 배너가 활성화되면, 페이지의 상단에 생성된 순서대로 표시됩니다. 명령줄에는 최신 배너만 표시됩니다.

배너는 배너를 해지 가능하도록 구성한 경우에만 닫을 수 있습니다.

## 알림 {#notifications}

GitLab은 페이지의 오른쪽 아래에 알림을 표시합니다. 알림에는 플레이스홀더가 포함될 수 있으며, 이는 현재 사용자의 속성으로 교체됩니다:

![이름 플레이스홀더를 사용하는 전체 알림 메시지 알림입니다.](img/broadcast_messages_notification_v17_7.png)

```markdown
{{name}}, would you like to give us feedback?
<a href="example.com">Take our survey!</a>
```

한 번에 두 개 이상의 알림이 활성화되면, 최신 알림만 표시됩니다.

알림은 다음의 플레이스홀더를 지원합니다:

- `{{email}}`
- `{{name}}`
- `{{user_id}}`
- `{{username}}`
- `{{instance_id}}`

사용자가 로그인하지 않은 경우, 사용자 관련 값은 비어 있습니다.

## 전제 조건 {#prerequisites}

관리자 액세스 권한이 있어야 합니다.

## 전체 알림 메시지 추가하기 {#add-a-broadcast-message}

GitLab 인스턴스의 사용자에게 메시지를 표시하려면 전체 알림 메시지를 추가합니다.

> [!warning]
> 전체 알림 메시지는 대상 지정 설정에 관계없이 API를 통해 공개적으로 액세스할 수 있습니다. 민감하거나 기밀 정보를 포함하지 마세요. 또한 전체 알림 메시지를 사용하여 특정 그룹이나 프로젝트에 비공개 정보를 전달하지 마세요.

전체 알림 메시지를 추가하려면:

1. 오른쪽 위 모서리에서 **Admin**을 선택합니다.
1. 왼쪽 사이드바에서 **메시지**를 선택합니다.
1. 오른쪽에서 **새로운 메시지 추가**를 선택합니다.
1. **메시지** 텍스트를 추가합니다:
   - 메시지 콘텐츠는 Markdown, 이모지, `a` 및 `br` HTML 태그를 포함할 수 있습니다.
   - `br` 태그는 줄 바꿈을 삽입합니다.
   - `a` HTML 태그는 `class` 및 `style` 속성을 허용하며 다음의 CSS 속성을 포함합니다:
     - `color`
     - `border`
     - `background`
     - `padding`
     - `margin`
     - `text-decoration`
1. **유형**에서 `banner` 또는 `notification`을(를) 선택합니다.
1. **테마**를 선택합니다. 기본 테마는 `indigo`입니다.
1. 사용자가 전체 알림 메시지를 닫을 수 있도록 하려면 **해지 가능**을 선택합니다.
1. 선택사항. Git 리모트 응답으로 명령줄에 전체 알림 메시지를 표시하지 않으려면 **Git 리모트 응답**을 선택 해제합니다.
1. 선택사항. 메시지를 사용자의 하위 집합에만 표시하려면 **대상 브로드캐스트 메시지**를 선택합니다:
   - 모든 페이지의 모든 사용자에게 표시합니다.
   - 특정 일치하는 페이지의 모든 사용자에게 표시합니다.
   - 그룹 또는 프로젝트 페이지에서 특정 역할이 있는 사용자에게만 표시합니다. 이 설정은 그룹, 하위 그룹 및 프로젝트 페이지에 메시지를 표시하지만, Git 리모트 응답에는 표시하지 않습니다.
1. 필요한 경우 전체 알림 메시지를 표시할 **대상 역할**을 선택합니다.
1. 필요한 경우 **대상 경로**를 추가하여 해당 경로와 일치하는 URL에만 전체 알림 메시지를 표시합니다. 와일드카드 문자 `*`를 사용하여 여러 URL을 일치시키고 경로를 지정합니다. 예를 들면:
   - `*/-/milestones`은(는) 모든 그룹 또는 프로젝트의 **마일스톤** 인덱스 페이지입니다.
   - `*/-/milestones/*`은(는) 개별 마일스톤 페이지만 해당합니다.
   - `*/-/milestones*`은(는) 인덱스 및 개별 마일스톤 페이지 모두 해당합니다.
1. 메시지가 시작하고 종료되는 날짜 및 시간(UTC)을 선택합니다.
1. **전체 알림 메시지 추가하기**를 선택합니다.

전체 알림 메시지가 만료되면, 더 이상 사용자 인터페이스에 표시되지 않지만 전체 알림 메시지 목록에 여전히 나열됩니다.

## 전체 알림 메시지 편집 {#edit-a-broadcast-message}

전체 알림 메시지를 변경해야 하는 경우 편집할 수 있습니다.

전체 알림 메시지를 편집하려면:

1. 오른쪽 위 모서리에서 **Admin**을 선택합니다.
1. 왼쪽 사이드바에서 **메시지**를 선택합니다.
1. 전체 알림 메시지 목록에서 메시지의 편집 버튼을 선택합니다.
1. 필요한 변경을 한 후에 **전체 알림 업데이트**를 선택합니다.

만료된 메시지는 종료 날짜를 변경하여 다시 활성화할 수 있습니다.

## 전체 알림 메시지 삭제 {#delete-a-broadcast-message}

전체 알림 메시지가 더 이상 필요하지 않으면 삭제할 수 있습니다. 활성화된 동안 전체 알림 메시지를 삭제할 수 있습니다.

전체 알림 메시지를 삭제하려면:

1. 오른쪽 위 모서리에서 **Admin**을 선택합니다.
1. 왼쪽 사이드바에서 **메시지**를 선택합니다.
1. 전체 알림 메시지 목록에서 메시지의 삭제 버튼을 선택합니다.

전체 알림 메시지가 삭제되면, 전체 알림 메시지 목록에서 제거됩니다.
